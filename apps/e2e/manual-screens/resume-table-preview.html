Passing case state

The test opens a table, drafts a menu item with a modifier, then resumes the table through the public current-order endpoint and asserts the same session, order, items, totals, and draft status.

Main flow

  • Cashier opens a shift on an active register.
  • An available table is opened into a live session.
  • A draft order is saved with one menu item and modifier.
  • User selects the occupied table again.
  • System loads the current open session and existing check state.

Observable result

  • Same session and order remain active on the occupied table.
  • Existing line item and modifier are returned intact.
  • Subtotal, tax, and total are preserved.
  • Service can continue from the same draft context.
